class AgentBuilder()

AgentBuilder can help user build an automatic task solving process powered by multi-agent system. Specifically, our building pipeline includes initialize and build. In build(), we prompt a gpt-4 model to create multiple participant agents, and specify whether this task need programming to solve. User can save the built agents' config by calling save(), and load the saved configs by load(), which can skip the building process.

buildâ€‹

def build(building_task: Optional[str] = None,
          default_llm_config: Optional[Dict] = None,
          coding: Optional[bool] = None,
          cached_configs: Optional[Dict] = None,
          use_oai_assistant: Optional[bool] = False,
          code_execution_config: Optional[Dict] = None,
          **kwargs)

Auto build agents based on the building task.

Arguments:

  • building_task - instruction that helps build manager (gpt-4) to decide what agent should be built.
  • default_llm_config - specific configs for LLM (e.g., config_list, seed, temperature, ...).
  • coding - use to identify if the user proxy (a code interpreter) should be added.
  • cached_configs - previously saved agent configs.
  • use_oai_assistant - use OpenAI assistant api instead of self-constructed agent.
  • code_execution_config - specific configs for user proxy (e.g., last_n_messages, work_dir, ...).
